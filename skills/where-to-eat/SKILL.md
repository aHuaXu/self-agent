# 就近用餐推荐

根据用户提供的两个位置和餐厅类型，推荐双方距离相近的用餐地点，并计算多种交通方式的到达时间。

## 适用场景

当用户有以下需求时，**必须调用此 skill**：
- "推荐个餐厅"
- "附近有什么好吃的"
- "帮我找吃饭的地方"
- "两个人约饭，找个中间位置的餐厅"
- "我在A地点，另一人在B地点，想吃X"
- 任何关于找餐厅、约饭、推荐美食的请求

## 前置依赖 ⚠️

**必须配置高德 MCP：**

1. 安装 MCP Server：
   ```bash
   npm install -g @amap/amap-maps-mcp-server
   ```

2. 配置 MCP（获取 API Key：https://console.amap.com/）：
   ```bash
   claude mcp add amap -e AMAP_MAPS_API_KEY=你的Key -- mcp-amap
   ```

3. 重启 Claude Code 使配置生效

## 输入解析

从用户输入中解析三个参数：
| 参数 | 说明 | 示例 |
|------|------|------|
| 地址A | 第一个位置（文字地址） | "建国路"、"望京" |
| 地址B | 第二个位置（文字地址） | "中关村"、"五道口" |
| 餐厅类型 | 火锅、川菜、日料等 | "火锅"、"川菜" |

**用户表达示例：**
- "我在建国路，另一人在中关村，找个吃火锅的地方"
- "推荐一个川菜馆，我在望京，对方在五道口"
- "帮我找日料，我在XXX，他在XXX"

## 执行流程

### Step 1: 地理编码

调用 `mcp__amap__maps_geo` 将地址转换为坐标：

| 参数 | 值 |
|------|-----|
| address | 输入的地址 |
| city | 城市名（可选，有助于提高准确性） |

返回的 `location` 字段格式为 "lng,lat"，取第一个结果。

### Step 2: 计算中点和半径

- **中点**：(lng1+lng2)/2, (lat1+lat2)/2
- **半径**：两点距离的一半（推荐范围 = 以双方连线为直径的圆）

### Step 3: 周边搜索

调用 `mcp__amap__maps_around_search` 搜索餐厅：

| 参数 | 值 |
|------|-----|
| keywords | 餐厅类型 |
| location | 中点经纬度（lng,lat） |
| radius | 搜索半径（米），设为 **半径的 2 倍**，最低 5000 米 |

**示例**：如果两点距离 20km，半径=10km，搜索半径设为 20000 米

### Step 4: 获取详情和到达时间

**默认只获取地铁时间，驾车时间为可选：**

1. `mcp__amap__maps_search_detail` — 获取评分（rating 字段）
2. `mcp__amap__maps_direction_transit_integrated` — **必须**计算地铁/公交时间
   - 参数：`{"origin": "餐厅坐标", "destination": "地址A坐标", "city": "城市名", "cityd": "城市名"}`
   - 再调用一次获取到地址B的时间

**可选：驾车时间**（只有用户明确要求时才调用）
- `mcp__amap__maps_direction_driving` — 计算驾车时间
  - 参数：`{"origin": "餐厅坐标", "destination": "地址A坐标"}`
  - 再调用一次获取到地址B的时间

**注意**：duration 返回的是秒，需要转换为分钟（除以 60，向下取整）

### Step 5: 排序输出

按"距离中点的距离"排序，优先推荐位置最均衡的餐厅。

## 输出格式

```
📍 推荐区域中心：中点坐标 (lng, lat)
📏 覆盖范围：以双方连线为直径的圆形区域

🍲 推荐餐厅（共 N 家）：

1. 餐厅名称
   距中点: Xkm | 距A: Xkm | 距B: Xkm |   地址: xxx ★X.X | 人均¥XX
   地铁：X分钟（从A出发）/ X分钟（从B出发）
   [驾车：X分钟（从A出发）/ X分钟（从B出发）]  ← 仅当用户明确要求时显示

2. 餐厅名称
   ...
```

**注意**：
- A 和 B 是用户输入的两个位置，输出中的"从A出发"指的是从第一个位置到餐厅的时间
- 默认只显示地铁时间，驾车时间只在用户明确要求时才显示

## 注意事项

- **至少展示 5 家餐厅**
- **默认只显示地铁时间**，只有用户明确要求（如"要驾车"、"开车"）时才额外显示驾车时间
- 如果用户只提供一个地址，以该地址为中心推荐
- 如果搜索结果为空，告知用户并建议扩大范围或换餐厅类型
- 如果 MCP 调用失败，检查 MCP 是否正确配置（环境变量 AMAP_MAPS_API_KEY）
- duration 返回的是秒，需要转换为分钟（除以 60，向下取整）
- 地铁时间取 transit API 返回的最快方案（duration 最小的那个）
